# Football Analysis 使用说明

## 启动

运行安装目录中的 `start_app.bat`，浏览器打开 `http://127.0.0.1:5050/`。

## 页面功能

左侧可选择期号和场次。对赛列表可勾选多场；勾选后，“采集全部对赛”“预测当前期全部对赛”“同步赛果并结算当前期”只处理勾选场次；未勾选时按当前期列表全量处理。
学习闭环区域包括“导入历史闭环”、“回放补齐历史期”、“训练学习候选”和“启用候选”。其中“回放补齐历史期”用于自动补入缺少的更早期号完整闭环样本。

系统会显示：普通胜平负推荐、让球盘推荐、盘口初盘/即时盘、主客覆盖率、EV、信心、比分预测、采集质量和赛后反馈。赛后结果与赛前预测对比会分开展示胜平负和让球盘的已结算、正确、错误、命中率和 ROI，避免把两个口径混成一个成功率。

“反馈概览”中历史反馈指标来自已经保存的 prediction runs 和 `feedback_logs`，反映旧策略和当前策略混合后的真实累计成绩；“当前策略回放”使用 active learning profile 对同一批已结算历史样本即时重算，用于查看当前策略自身是否达标。

## 让球盘推荐口径

让球盘推荐直接显示为“主推主队”“主推客队”“轻仓主队”“轻仓客队”或“观望”。盘口区域只显示盘口本身，例如“主队受让半球”或“主队让球半”。

让球盘计算使用 500 亚盘页的平均初盘和即时盘，结合采集到的球队实力、近期状态、主客表现、伤停阵容、赛程动机、赔率和投注热度等特征进行覆盖率和 EV 评估。

“胜平负建议仓位”对应普通胜平负主推荐动作；让球盘推荐卡片中的“建议仓位”按让球盘推荐侧、覆盖率、赔率、EV 和信心临时计算展示。两者都使用分数 Kelly 口径，不是 full Kelly 直接下注。

## 历史数据与回测

安装包携带当前历史 SQLite 数据库。历史期数已包含胜平负预测、让球盘推荐、赛果同步和回测字段。新环境打开后即可查看已有期数。

如果训练提示完整已结算期不足，先点击“回放补齐历史期”。系统会按缺口向更早期号回放：同步对赛、采集赛前字段、预测整期、同步赛果、写入反馈。只有整期反馈齐全的期号才算完整学习闭环样本；部分闭环或采集失败的期号需要修复后重跑。

推荐顺序：回放补齐历史期 -> 训练学习候选 -> 达标后启用候选 -> 再预测新一期。

当前 active learning profile 为 #45，策略类型 `handicap_bucket_table`，用于让球盘推荐。当前全量历史回放样本 1232 场，执行 964 场，执行率约 78.3%，执行命中率约 74.3%。它不会自动改写历史单场页面中已经保存的旧 prediction run；若要让历史单场详情也显示新策略结果，需要重新生成对应预测。

## 更新新一期

通常流程为：同步当前对赛，采集全部对赛，预测当前期全部对赛。比赛结束后点击同步赛果并结算当前期，系统会更新命中率和 ROI。

## 配置

进入“系统配置”页面或直接编辑 `.env`。修改后重启应用生效。
